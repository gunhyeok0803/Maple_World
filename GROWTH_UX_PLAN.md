# 성장 UX 개선 — 확정 계획 (인벤/강화/승급/컬렉션)

> 작성 2026-07-13. 3축 조사(코드/HTML/레퍼런스) 종합 + 사용자 결정 반영. 구현은 **항목별 단계 진행**(UI→기능→검증, 400 방지). 항목6 연출은 **최후 보류**.

## 조사 확정 사실 (추측 금지 근거)
- **단일 소스 정상** [코드]: 모든 성장 시스템/UI가 하나의 `InventoryController`(@Sync CSV)만 참조. 장비 데이터 복제 없음. 진짜 stale 위험 = 인벤 상세팝업이 슬롯 index로 물림(재정렬 시 오참조).
- **2축 등급(사용자 확정)**: `_ItemGrade`(내부 Rare~Legendary, **표시 하급/중급/상급/최상급**, 승급축) · `_PotentialGrade`(내부 Rare~Legendary, **표시 레어/에픽/유니크/레전더리**, 잠재축). HTML §01은 하급~최상급 미사용이라 했으나 **사용자가 의도적으로 하급~최상급 표시 확정**(메이플키우기 기준) → 재론 안 함.
- **장착=가방과 분리 풀** [코드]: `EquipInternalEx`가 장착 시 가방 -1 후 페이퍼돌 이동. 장착품은 가방에 없음. (사용자 전제 "장착품이 인벤에 남는다"와 반대.)
- **승급=결정론적 병합** [코드+HTML §05]: 레어800→에픽 / 에픽200→유니크 / 유니크20→레전드리. **성공확률·실패·비용(메소) 없음.**
- **컬렉션=등급별 반복 등록** [코드+HTML §05]: 등록 정상 동작(소비→BookCsv→저장). 구조 Lv10/20/30 × 부위. **HTML에 Lv200 도감/진행률 개념 없음.** [레퍼런스] 메이플 몬스터컬렉션(실루엣+%+지역)은 사냥 자동등록형이라 성격 다름.

## 사용자 결정
- **항목3 = 표시 통합**: 강화창은 가방+장착을 **한 스크롤 목록**으로(장착품도 강화 가능), 스타포스는 그 목록에서 **레전드리(최상급)만**. 승급/컬렉션은 가방 스택만.
  - mod1: 착용 4칸이 너무 적음 → **27슬롯 규모를 인벤처럼 스크롤**로. (통합 목록이 스크롤로 다 수용)
  - mod2: 착용/인벤 **텍스트 크기 통일**(현재 강화창 "장착중인 장비"18 vs "인벤 장비"14).
- **항목5 = 등록 동작 점검만** (UI 대개편 보류).

## 항목별 구현안 (현 구조 유지, 필요한 것만)
1. **인벤 선택 해제**: ❌ **취소**(2026-07-13 사용자 "1번 진행 x, 착각"). 되돌림.
2. **등급/수량 표시 일관화 (사용자 재정의)**: 모든 아이템 셀을 인벤과 동일 구성으로 — **색=잠재등급(레어~레전더리) 슬롯 + 텍스트=아이템등급(하급~최상급) QBadge + 수량(스택>1일 때만, 자연히 승급/인벤에만)**. 격차였던 **강화·승급 그리드 CellTemplate에 QBadge 추가**(인벤 QBadge 3컴포넌트 복제) → `UIEquipmentSlotWidget.SetData`가 자동 채움. ✅ 강화 그리드 "하급" 배지 표시 검증. 파일: ui/EnhanceGroup.ui, ui/PromoteGroup.ui.
3. **표시 통합 + mod1/2**: 강화창 좌패널의 고정 착용 4칸 제거 → 단일 스크롤 그리드에 **가방+장착 병합**(장착품엔 "장착" 배지). 스타포스 탭은 레전드리 필터. 텍스트 크기 통일. 파일: UIEnhancePanel(BuildList/EnsureListWired), UIEquipmentSlotWidget(배지), ui/EnhanceGroup.ui(좌패널 정리·헤더 폰트). 승급/컬렉션도 가방 스택 기준 동일 스타일.
4. **승급 UI**: 현재등급(하급~최상급)→승급후등급(색 배지), 필요 수량 N/보유 수량, 결과(1개+일괄 가능 횟수), 소모 경고. **성공확률·비용 없음**(결정론적). 파일: UIPromotePanel, ui/PromoteGroup.ui.
5. **컬렉션 점검**: 등록/차감/저장/표시 정상성 런타임 검증. UI 개선 보류. 파일: (검증만) CollectionSystem/UIPromotePanel.
6. **연출**: 최후 보류.
