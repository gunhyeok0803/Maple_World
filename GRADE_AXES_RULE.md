# 등급 2축 규칙 (절대 준수 — 사용자 강한 지침)

두 등급은 **완전히 독립된 서로 다른 축**이다. 절대 붙여서(예: "파랑 하급") 표현하거나 혼동하지 말 것.

---

## enum 키 이름 (충돌 금지)

| 축 | 정수 | enum 키(영문) |
|---|---|---|
| 잠재등급 `_PotentialGrade` | 1~4 | **Rare / Epic / Unique / Legendary** |
| 아이템등급 `_ItemGrade` | 1~4 | **Low / Normal / High / Best** |

- 두 enum의 키 이름은 **절대 겹치지 않는다.** `_ItemGrade.Rare` 같은 이름 금지(과거 충돌 → Low/Normal/High/Best로 개명, 정수값 1~4 불변).
- StackKey/CSV는 정수(1~4)로 저장하므로 개명해도 저장데이터 호환.

---

## 축 1) 잠재등급 (PotentialGrade) = **슬롯 배경 색상**

| 잠재등급 | 색상 |
|---|---|
| 레어(Rare) | 파랑 |
| 에픽(Epic) | 보라 |
| 유니크(Unique) | 금 |
| 레전드리(Legendary) | 초록 |

- 렌더: `_PotentialGrade:SlotRUID(pg)` — 슬롯 배경 이미지 색.
- **업그레이드 경로: 잠재(큐브) 탭에서 증가·업그레이드 가능.**
- 데이터 필드: `pg`.

## 축 2) 아이템등급 = **색 없는 텍스트 배지**

| 아이템등급 | 표시 |
|---|---|
| 하급 → 중급 → 상급 → 최상급 | 색 없는 **텍스트**(QBadge) |

- 렌더: `_ItemGrade:Name(grade)` — plain 텍스트, **색 입히지 말 것**.
- **업그레이드 경로: 승급 탭에서 아이템등급 상승.**
- 데이터 필드: `grade`.

---

## 핵심 (혼동 금지)

- **색상 = 잠재등급**, **텍스트 = 아이템등급**. 끝. 서로 무관.
- 예: **하급 품질 + 유니크 잠재** = **금색 슬롯 + "하급" 텍스트**. (색이 하급을 뜻하지 않음)
- "파랑=하급" 같은 표현·구현 절대 금지. 파랑은 잠재 레어일 뿐이다.
- 모든 UI(인벤토리·컬렉션·강화·승급·잠재·환생)는 공통 위젯 `UIEquipmentSlotWidget` 하나로 이 규칙을 동일 적용한다. `itemPg` = 실제 잠재등급(고정값 금지), `grade` = 실제 아이템등급.
