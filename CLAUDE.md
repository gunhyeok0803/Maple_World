# 작업 진행 규칙 (사용자 지정)

- **한 번에 몰아서 진행하지 말 것 (400 에러 방지).** 큰 작업은 작은 단계로 쪼개서 진행하고, 각 단계마다 결과를 확인한 뒤 다음 단계로 넘어간다.
  - 한 응답(턴)에서 대량의 파일 편집·대형 도구 입력·긴 스크립트를 몰아넣지 않는다.
  - 빌드/UI 스크립트는 한 번에 하나씩 실행하고 로그로 확인 후 다음으로.
  - 다단계 작업은 진행 전 사용자와 상의하고, 단계별로 승인/확인을 받는다.
